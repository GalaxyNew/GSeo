## 一、数据库 Schema（**PostgreSQL 16** + Prisma，广州节点单库）

> v1.1 修订（2026-07-08）：广州已运行 PostgreSQL16 + Redis7（Docker，seo-postgres），弃用 SQLite 方案直接用 PG——多写入方场景更稳，Redis 留作后续队列/缓存。

```prisma
// —— M1 计划中枢 ——
model Plan {            // 计划树：年→月→日 三层
  id        String @id            // PL-2026 / PL-2026-07 / PL-2026-07-08-01
  parentId  String?
  level     String                // year | month | daily
  title     String
  status    String                // planned | active | done | dropped
  ownerId   String                // agent-id
  taskId    String?               // 关联任务卡 T-xxx（贯穿主键）
  items     PlanItem[]
}
model PlanItem {        // 计划条目（分类管理）
  id        String @id            // PI-xxxx
  planId    String
  category  String                // keyword | backlink | article | tech-seo
  title     String
  detail    String?               // 关键词词组/外链目标/文章主题等
  status    String                // todo | doing | review | done | blocked
  ownerId   String
  taskId    String?
  metrics   String?               // JSON：目标值/当前值（如排名、收录）
}

// —— M2 发布中心（复用已验收 P0 对象模型）——
model ReleasePlan {
  id        String @id            // RP-20260708-01
  title     String
  status    String                // draft | pending_approval | approved | executing | done | rolled_back | rejected
  createdBy String
  taskId    String?
  items     ChangeItem[]
  backups   BackupRecord[]
}
model ChangeItem {
  id           String @id
  releasePlanId String
  seoClass     String             // batch_ok | manual_confirm | blocker（已验收三级分类）
  uiState      String             // normal | blocked | audit（三态）
  path         String             // 变更目标（文件/配置/内容）
  diff         String             // 变更内容描述或 patch 摘要
  decision     String?            // approved | rejected（仅用户可写）
  decidedAt    DateTime?
}
model BackupRecord {   // 铁律：执行前必有
  id            String @id
  releasePlanId String
  location      String            // 备份路径（生产 VPS 上）
  sha256        String
  retainUntil   DateTime          // 默认 +30 天
}
model RollbackPlan {
  releasePlanId String @id
  steps         String            // 可执行回滚步骤
  verified      Boolean           // dry-run 验证过
}

// —— M4 事件账本（统一事件 ID）——
model Event {
  id        String @id            // EV-20260708-0001
  ts        DateTime
  actor     String                // agent-id | user
  action    String                // sync | analyze | propose | approve | execute | alert | rollback | note
  taskId    String?
  input     String?               // 摘要，敏感信息用 [REDACTED]
  output    String?
  evidence  String?               // artifact 路径 / 飞书 msg_id / URL
}

// —— M3 指标（大屏数据源）——
model MetricDaily {
  date      String                // 2026-07-08
  source    String                // gsc | dataforseo | site-health | visitors
  key       String                // clicks / impressions / position / kw:{词} / uptime ...
  value     Float
  meta      String?               // JSON
  @@id([date, source, key])
}
```

## 二、artifact 契约（采集→消费唯一接口）

```json
{
  "schema_ver": "1.0",
  "source": "gsc | dataforseo",
  "property": "sc-domain:igoriptv2.com",
  "date_range": ["2026-07-07", "2026-07-07"],
  "generated_at": "2026-07-08T05:10:00+08:00",
  "sha256": "<payload 的哈希>",
  "payload": { "rows": [ ... ] }
}
```
规则：Ingestor 先验 sha256 与 schema_ver，不合格拒收并写 Event(action=alert)；payload 结构变更必须升 schema_ver 并同步双端。

## 三、Agent API 契约（token 鉴权，最小权限）

| 端点 | 方法 | 谁可调 | 说明 |
|---|---|---|---|
| /api/events | POST | 全体 agent | 写事件（自动生成 EV-ID） |
| /api/plans?level=&status= | GET | 全体 agent | 读计划树 |
| /api/plan-items/{id} | PATCH | 条目 owner | 更新自己条目的 status/metrics |
| /api/release-plans | POST | agent 提案 | 创建 draft（不能自批） |
| /api/release-plans/{id}/decision | POST | **仅用户** | 审批动作（后台 UI 触发） |
| /api/metrics/daily | POST | Ingestor | 指标入库 |

红线：agent 无 decision 权限；所有写操作自动附 Event；token 按 agent 分发、可吊销。

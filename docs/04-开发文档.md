## 2. 技术栈与目录

Next.js（App Router）+ Prisma + PostgreSQL 16 + Caddy。仓库结构见治理规范第二节；应用在 `apps/command-center/`。

```bash
# 本地/广州开发起步
cd apps/command-center
cp .env.example .env        # DATABASE_URL=postgresql://...@127.0.0.1:5432/seo
npx prisma migrate dev
npm run dev
```

## 3. 开发流程（强制）

1. 领 issue（有里程碑+assignee）→ 任务账本对应卡领单（卡头 github_issue 字段对齐）
2. 分支 `feat/T{卡号}-{短名}`，小步提交
3. 自测清单（沉淀自已验收标准）：`git status` 干净 / `npx prisma validate` / 冒烟脚本 / `npx eslint` / `git diff --check`
4. PR：模板必填改动清单+验证输出；code-reviewer 门禁；Hermes 合并
5. 合并后：部署广州 → 更新 CHANGELOG → 关 issue（贴证据）→ 群通知
6. 涉及西班牙站：PR 里先贴 backup.sh 执行证据，否则 reviewer 直接打回

## 4. 部署（广州）

```bash
# 应用（首次由 DevOps 配 systemd/pm2，之后 CI 或手动）
cd /opt/seo-system/app && git pull && npm ci && npx prisma migrate deploy && npm run build && pm2 reload gseo
# Caddy 反代 :80 → next :3000（静态 v0 板块随模块上线逐步移除）
curl -s http://127.0.0.1/dashboard -o /dev/null -w '%{http_code}\n'   # 期望 200
```

回滚：`pm2 reload gseo --update-env` 回退上一 release 目录（DevOps 维护 releases/ 双目录切换）；数据库迁移回滚用 prisma migrate 的 down 脚本，上线前必须提供。

## 5. 数据管道

- Collector（法国）：cron 拉 GSC/DataForSEO → artifact JSON（契约第二节）→ HTTPS 推广州
- Ingestor（广州）：验 sha256/schema_ver → MetricDaily 入库 → 失败写 Event(alert)
- sync-task-ledger.sh：任务账本 → all.json →（M1 后改为直接写 PlanItem/Event API）

## 6. 约定

- 事件必写：每个 agent 完成动作调 `POST /api/events`（治理规范映射铁律）
- 时区统一 GMT+8；日期字符串 `YYYY-MM-DD`
- 提交信息：`feat(T20260708-10): 一句话`；中文
- 任何"完成"的定义 = PR 合并 + 部署可访问 + CHANGELOG + issue 关闭，四者缺一不算
